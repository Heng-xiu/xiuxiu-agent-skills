# S3 + S4 · 觀點併入 + 多平台寫作 + 自動修正 gate

> 按需載入：S3（併入你的核心觀點）、S4（寫成各平台貼文）。本檔最重要的是 **S4 的自動修正循環**——這是「丟下去不斷修正到可以發文」的引擎。

---

# S3 · 觀點併入

把使用者「想在文章裡闡述的核心觀點」焊進 S1 選定方向 + S2 爆款結構。三者缺一不可：
- **角度**（S1）= 切入點
- **爆款結構**（S2）= 怎麼排會有人看
- **你的觀點**（S3）= 為什麼是你寫、跟別人不同

## 步驟

1. **取得使用者原話觀點**：問「這篇你最想讓讀者記住/相信的一句話是什麼？」用**他的原話**記在狀態檔，別改寫掉他的本意。
2. **找張力**：他的觀點 vs 主流看法的衝突點在哪？沒有張力的觀點 = 沒人想轉。把它逼成反命題：「大家以為 X，但我認為 Y，因為 Z」。
3. **焊接**：把觀點塞進選定的 hook 骨架。產出**一句話 thesis** + **一個反命題 framing**，這是所有平台草稿的共同核心（各平台只是包裝不同）。

## 輸出（寫回狀態檔 S3）

```
- 我的核心觀點（原話）: 「…」
- 張力點: 主流說 X，我說 Y
- 焊接 thesis（一句話）: 
- 反命題 framing: 「大家以為…，其實…」
```

**規則**：觀點是使用者的，不是你的。你只負責放大張力、找最利的 framing，不替他換立場。

# S4 · 多平台寫作 + 自動修正 gate

## 0. 寫之前必做

1. **讀語氣檔**：`social-post/style_profile.md`。不存在 → 請使用者貼 5-10 篇他自己的貼文，現場提煉語氣（句長、口頭禪、emoji 習慣、人稱、收尾方式），暫存。**公式 < 語氣**：寧可結構鬆一點，也不能不像他講話。
2. **選公式**：按目標平台，從 `social-post/references/formulas.md` 挑對應公式（下方對照表）。
3. **逐平台分別生成**：每平台重寫，**絕不照搬同一段**。

## 1. 平台 × 公式 × 角度 對照（從 social-post formulas.md 取）

| 平台 | 主吃 | 預設公式（social-post） | 角度適配 |
|---|---|---|---|
| **FB** | 陌生人分享 + 留言 + dwell | F6b 純血 4段4句 / F22 脆弱 confess / F23 行業洞察 / F25 集體 | 長敘事、meta 鉤子、留言 CTA |
| **Threads** | 轉發 + keyword | F19 立場宣言（1段不換行）/ F7 POV 吐槽 | 短站隊、敵人英雄、純粹動機 |
| **IG** | 儲存 + Reels 完成率 | F2 截圖先丟 / F18 Reels / carousel 化 | 首圖 hook、清單、framework |
| **X** | Reply + 收藏 + 完讀 | F5 thread 對打 / F8 外部標記 / Cole 7 模板 | Hook 推 + 1/3/1 + TL;DR + CTA |
| **小紅書** | 封面標題 + 收藏 | （無 social-post 公式）標題黨 + 圖文 + 乾貨/種草 + # 標籤 | 數字標題、痛點、emoji、結尾標籤 |

> 小紅書沒有現成公式：用 S2 挖到的小紅書爆款卡當骨架（數字標題 + 痛點開頭 + 條列乾貨 + emoji + 5-10 個 # 標籤）。

## 2. 🔁 自動修正 gate（核心：丟下去改到達標）

每篇草稿生成後，**自動**跑下列 gate。**不過自動重寫，循環到通過，達標才停**——不要每改一次就停下來問，連續通過所有 gate 才把成品給使用者。

```
生成草稿
  → 跑 GATE（下方清單，逐條 pass/fail）
  → 有 fail？ → 標出哪條 fail + 為什麼 → 自動重寫該部分 → 回到 GATE
  → 全 pass？ → 停，輸出待發版 + 標註它通過了哪些 gate
  → 連跑 3 輪仍有同一條 fail → 停，告訴使用者「這條卡住，可能是素材/觀點問題」，給選項
```

### GATE 清單（每篇都跑）

**A. 語氣 gate**（最高優先，fail 必重寫）
- [ ] 讀起來像 `style_profile.md` 的人在講話？（句長、口頭禪、emoji、人稱、收尾）
- [ ] 沒有 AI 腔（沒有「在當今這個…」「總而言之」「讓我們一起」這種非他的套話）

**B. 手機可讀 gate**（見 SKILL.md 手機優先）
- [ ] 首屏前 1-2 行能止滑（手機只看得到這麼多）
- [ ] 每段 ≤ 3 行手機螢幕；長段已拆
- [ ] 5 秒讀懂：meta 層 ≤ 2、一段數字 ≤ 3 個

**C. 平台結構 gate**（按平台）
- [ ] FB：4 段 4 句（用 F6b 時）/ 留言 CTA 非邀請碼
- [ ] Threads：1 段不換行、連續逗號流、「！」≤「！！」、60-150 字
- [ ] IG：首圖 hook 文案到位 / carousel 第 1 張止滑
- [ ] X：Hook 推回答了「給誰看/講什麼/為何信你/得到什麼」
- [ ] 小紅書：標題有數字+痛點+結果、結尾有 5-10 個 # 標籤

**D. Viral 4 條件 gate**（FB/社群長貼，取自 social-post）
- [ ] 結構（對應公式骨架）✓
- [ ] 純血 voice（像他講話）✓
- [ ] 全新敘事意圖（這個 framing 近期沒重複，避免 R5 主題冷卻）✓
- [ ] 黃金時段（時段建議留給 S5）✓
> 4 個 AND，任一缺 = 死。

**E. 硬規則 gate**（fail 直接擋，不可發）
- [ ] 🚨 FB/Threads 正文**無外部連結**（連結移留言區）
- [ ] 觀點沒被改掉（還是使用者 S3 的原意）
- [ ] 跨平台是分別生成、非複製貼上

### 重寫時的判斷
- 語氣 fail → 回去重讀 style_profile，逐句調成他的口吻
- 停滑 fail → 換更強的 hook 骨架（從 S2 爆款卡挑另一個）
- 敘事意圖重複 fail → 換角度 framing（不是換詞）
- 連 3 輪卡同條 → 通常是 S2 素材不夠或 S3 觀點不夠利，回上一關，別硬改

## 3. 輸出格式（S4 待發版）

```markdown
# S4 多平台草稿: {thesis}

## FB
{草稿}
— 公式 {Fxx} | 通過 gate: 語氣✓ 手機✓ 結構✓ viral4✓ 硬規則✓ | 修正 {N} 輪

## Threads
{草稿}
— 公式 {Fxx} | gate ...

## IG
{首圖文案 + carousel 各張 / 或 Reels 腳本}
— gate ...

## X
{thread 逐條}
— gate ...

## 小紅書
{標題 + 正文 + # 標籤}
— 骨架 {小紅書爆款卡 N} | gate ... | ⚠️ 需手動發

## 待確認
逐平台問使用者要不要調，確認後 → S5 發文建議
```

## 規則
- **語氣優先於公式**：不像他講話，其餘都白搭，語氣 gate fail 一律重寫
- gate 自動跑到全 pass 才交件，別每輪都打斷使用者
- 連 3 輪卡同一條 → 停下說清楚，別假裝改好了
- 每平台分別生成；硬規則 gate（無外連）不可 bypass
- 通過哪些 gate 要標出來，讓使用者知道這篇「為什麼可以發」
