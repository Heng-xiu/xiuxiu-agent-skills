# S1 · 市場性審視 + 提案方向

> 按需載入：S1（使用者丟一個粗淺 idea）。目標 = 判斷這題在**社群**值不值得寫，並給 3-5 個可選提案方向（排序），最後決定走短貼還是長文。

## 這一關要回答三件事

1. 這個 idea 有沒有**社群市場性**？（不是學術價值，是「有人會停下來看、會互動、會分享」）
2. 從這 idea 能拆出哪幾個**提案方向**？哪個最強？
3. 這題該寫成**短貼文**，還是值得走**長文鏈**？

---

## 一、社群市場性 6 維評分（每維 1-10，加權）

改寫自 longform-filter 的評分表，但換成**社群短貼**的成功要素（長文要深度，社群要「停 + 互動 + 分享」）。

| 維度 | 權重 | 社群版意義 | 低分訊號 |
|---|---:|---|---|
| **停滑力 Stop Power** | 2.5 | 首屏 1-2 行能不能讓人拇指停下（好奇/反差/情緒/利益） | 平鋪直敘、要讀三行才知道在講啥 |
| **共鳴/身分 Identity** | 2 | 讀者會不會覺得「這就是在說我」→ 想留言/轉給朋友 | 太小眾、與讀者無關、純自說自話 |
| **互動鉤 Engagement Hook** | 2 | 有沒有天然的留言/分享/儲存動機（提問/清單/索取/站隊） | 看完無話可說、沒有下一步 |
| **觀點張力 Tension** | 1.5 | 有沒有反命題、衝突、反直覺、可站隊的立場 | 正確但無聊、人人都同意 |
| **你的可信度 Credibility Fit** | 1 | 你有沒有資格/經歷講這題（避免空泛） | 你沒做過、只是轉述 |
| **素材可得性 Material** | 1 | 找得到爆款範本 + 你有料可填（數據/案例/經歷） | 抽象、無例子、無數據 |

**總分門檻**：
- `8.0–10`：強發（Strong Go）
- `7.0–7.9`：能發，先磨角度
- `5.5–6.9`：可能，需縮窄角度或補料
- `<5.5`：殺掉，或降級成一句吐槽 / Mode A 日常

打分時**扣分**：泛泛 tips、純蹭熱點、與讀者無關、沒有任何互動動機、其實是私人日記。

## 二、提案方向生成（從一個 idea 拆 3-5 個角度）

用 **4A 矩陣**（取自 x-mastery/writing-workshop.md）把一個主題拆成多角度。一個 idea × 4 角度 = 一批提案：

| 角度 | 問句 | 適合 |
|---|---|---|
| **Actionable 可操作** | 「怎麼做 X」具體步驟/框架/清單 | 儲存型、工具型受眾 |
| **Analytical 分析** | 「為什麼 X 會這樣」拆機制/數據 | 思考型、行業受眾 |
| **Aspirational 願景** | 「X 之後你能變成怎樣」 | 情緒、身分認同 |
| **Anthropological 觀察** | 「大家都在 X，這現象代表什麼」 | 共鳴、站隊、討論 |

再疊一層**社群特化角度**（短貼專屬，比 4A 更直接爆）：

- **反命題**：「大家以為 X，其實 Y」（張力最高）
- **個人實測翻車**：「我用 X 結果…，三件沒人說的事」
- **脆弱 confess**：「我一直卡在 X，直到…」（broke 鐵粉圈神器，見 social-post F22/F27）
- **稀缺公開**：「我把私藏的 X 開源了」（儲存 + 分享）
- **里程碑 + 集體**：「我們一起做到了 X」（反個人主義，broke +30-50%）

### 每個提案方向產出

```
方向 N：{一句話角度}
- 4A/特化類型: {Actionable / 反命題 / 脆弱 confess ...}
- 目標讀者: {誰會停下來}
- 預期效果: 拉新 / 留人 / 引發討論 / 儲存 / 分享
- 預期最強平台: {FB 長敘事 / Threads 短站隊 / IG 圖文 / X thread / 小紅書 圖文}
- 停滑力預估: X/10
- 需要的素材: {爆款範本 + 你的料}
```

把所有方向**排序**，標出推薦的 1-2 個，讓使用者選或改（檢查點）。

## 三、短貼 vs 長文 岔路判斷

不是每個 idea 都該寫成爆款短貼。判斷門檻：

**走長文鏈**（`longform-collect → filter → narrative → outline`）當且僅當：
- 證據密度高（有 primary source / 數據 / 多個案例）**且**
- 能撐 1500 字以上不灌水 **且**
- 有可持續的張力（中段不會塌）**且**
- 讀者有「持久的問題」要被回答（不是一時熱點）

**走短貼**（直接 S2→S5）當：
- 一個鉤子 + 一個觀點就講完
- 靠情緒/反差/站隊驅動，不靠論證厚度
- 蹭時效熱點

**兩者都做**（推薦給有料的題）：
- 先走長文鏈產母體 → 回 S4 原子化成 IG carousel / FB 長文 / X thread / 小紅書 多篇
- 長文 = 資產（SEO / newsletter / 深度信任），社群貼 = 流量入口

判斷後寫回狀態檔 `S1 → 短貼 vs 長文` 欄。

## 四、目標平台確認

S1 末尾，若使用者沒指定平台，問一句：
> 「這題你想發哪些平台？FB / Threads / IG / X / 小紅書？（可複選，我會各自適配）」

寫回狀態檔 `目標平台`。不同平台吃不同角度（見 platform-matrix.md），會影響 S2 挖哪些平台的爆款、S4 用哪些公式。

## 輸出格式（S1 報告）

```markdown
# S1 市場性審視: {idea}

## 市場判斷
總分 X/10 → {Strong Go / Go-sharpen / Maybe / 短貼 / Kill}
一句話理由:

## 6 維評分
| 維度 | 分 | 理由 |
|---|---|---|
| 停滑力 | /10 | |
| 共鳴身分 | /10 | |
| 互動鉤 | /10 | |
| 觀點張力 | /10 | |
| 可信度 | /10 | |
| 素材可得 | /10 | |

## 提案方向（排序）
1. ⭐ {方向} — 停滑 X/10 — {理由} — 最強平台 {X}
2. {方向} — ...
3. ...

## 短貼 vs 長文
{判斷 + 理由}

## 建議下一步
選一個方向 → 進 S2 挖素材。目標平台: {…}（未定請使用者確認）
```

## 規則
- 不因為「熱門」就給高分；要問「有人會停 + 會互動嗎」
- 一個 idea 至少給 3 個方向，但**推薦聚焦 1-2 個**，別給一堆模糊可能
- 若最強產出其實只是一句吐槽，直說「這題寫成短貼/日常就好，不值得一條龍」
- 評分要短理由，不要空話
- 決策明確：Go / Maybe / 短貼 / Kill，寫回狀態檔
