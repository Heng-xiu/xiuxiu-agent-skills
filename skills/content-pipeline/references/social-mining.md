# S2 · 社群爆款素材挖掘

> 按需載入：S2（使用者要找素材 / 找爆款 / 看別人怎麼寫）。這是本 skill **新增的核心能力**，現有 skill 都沒有。

## 先分清兩種「素材」（別搞混）

| | **爆款範本素材**（這一關） | **事實證據素材**（longform-collect） |
|---|---|---|
| 找什麼 | 高流量**貼文**：hook 句式、結構、格式、互動數據 | 數據、論文、案例、官方來源 |
| 為了 | 學「怎麼寫會爆」「什麼角度有人看」 | 建論證、撐長文、查證 |
| 工具 | WebSearch 各平台爆款 | longform-collect 多輪檢索 |
| 何時叫 | S2 預設 | 只有當這篇**需要事實/數據**時，平行叫 longform-collect |

**判斷**：純情緒/觀點短貼 → 只做爆款挖掘。需要數據/案例佐證（行業分析、實測、科普）→ 兩個都做，把 longform-collect 的 Material Cards 也寫進狀態檔 S2。

---

## 一、各平台爆款搜尋法

對「選定方向 + 目標平台」逐平台搜。目標：每平台找 3-5 個**真實高互動**貼文，拆出可借用的 hook/結構。

### 通用搜尋骨架（WebSearch）

```
{主題關鍵字} {平台名} 爆款 / 高讚 / 熱門
{主題關鍵字} viral {platform}
{主題關鍵字} {平台} 範例 / 範本
site:{平台域名} {主題關鍵字}        # 用 Google 索引繞登入牆
"{主題}" 万赞 / 10万+ / 爆文          # 中文平台數字訊號
```

### 小紅書（圖文筆記，中國平台）⚠️ 見邊界段

```
小红书 {主題} 爆款笔记
小红书 {主題} 万赞
{主題} 小红书 标题 公式
site:xiaohongshu.com {主題}
```
要抽：**標題黨句式**（數字 + 痛點 + 結果，如「我用XX，30天瘦了X斤」）、emoji 密度、結尾標籤（#）、首圖封面文案、「乾貨 vs 種草」類型。
小紅書吃：**封面標題 + 圖文 + 收藏**。標題決定 70% 點擊。

### Threads（短句高回覆/轉發）

```
Threads {主題} viral
{主題} threads 高互動
```
要抽：**短句（<150 字）**、連續逗號流、站隊/立場宣言、提問結尾。Threads 吃**轉發 + keyword 觸發**（見 social-post F19）。

### IG（圖卡/Reels）

```
instagram {主題} carousel viral
{主題} reels 高觸及
{主題} instagram 帖 高赞
```
要抽：**首圖 hook 文案**、carousel 第 1-2 張的鉤子、Reels 前 3 秒、儲存型結構（清單/framework）。IG 吃**儲存 + Reels 完成率**。

### FB（長敘事高分享）

```
facebook {主題} 爆文 高分享
{主題} facebook post viral
```
要抽：**4 段 4 句結構**、meta 鉤子、脆弱 confess、集體 framing、留言 CTA（不附連結）。FB 吃**陌生人分享 + 留言 + dwell time**（分享 = 20× 讚，見 social-post evaluation.md）。

### X / Twitter（thread）

```
{topic} twitter thread viral
{topic} X thread 高互動
```
要抽：**Hook 推**（好奇缺口 + 可信度錨點 + 具體交付）、1/3/1 節奏、每條獨立成文、TL;DR + CTA。X 吃**Reply（27× Like）+ 收藏 + 完讀**。

## 二、爆款範本卡（每個爆款抽成一張卡）

對每個找到的高互動貼文，**不要只貼連結**，抽成可複用的卡片：

```markdown
### 爆款卡 N
- 平台: {小紅書/Threads/IG/FB/X}
- 來源: {url 或 帳號 + 日期}
- 互動數據: {讚/收藏/分享/留言/瀏覽，有就填}
- 主題關聯: {跟你的方向哪裡相關}
- 🎣 Hook 句式: 「{把開頭抽象成可填空骨架}」
  例: 「我用＜工具＞＜時間＞，＜反差結果＞。但有三件事沒人說：」
- 🏗️ 結構: {段落數 / 節奏 / 視覺切分方式}
- 💥 為何爆: {停滑點 + 互動點 + 分享動機，1-2 條}
- ♻️ 可借用點: {你能搬走的句式 / 結構 / 角度，不是抄內容}
- ⚠️ 不可照搬: {綁原作者人設/數據的部分}
```

**抽卡原則**：偷**結構與句式**，不偷內容。把具體爆款反向工程成「填空骨架」，這樣能套你自己的料。

## 三、跨平台共同模式（挖完後的綜合）

找完 3-5 平台後，做一次綜合，回答：
- 這個主題在哪個平台**最容易爆**？（決定主戰場）
- 反覆出現的 **hook 句式** 是哪幾種？（提煉 2-3 個最強骨架給 S4 用）
- 哪些角度**已經被寫爛**（避免）/ 哪個角度**還沒人好好寫**（你的縫隙）？
- 各平台**格式差異**（同主題在 Threads 要短站隊、FB 要長敘事、IG 要圖、小紅書要標題黨）

寫回狀態檔 S2 的「跨平台共同模式」。

---

## 四、⚠️ 小紅書誠實邊界

- **挖料**：可。用 WebSearch + `site:xiaohongshu.com` + 第三方彙整繞登入牆，抽標題/結構/互動量級。但深層內容常被登入牆擋，量級數字未必精準 → 標「概估」。
- **若搜不到夠多**：請使用者**直接貼 3-5 篇**他看到的小紅書爆款截圖/文字，現場拆卡（最可靠）。
- **發佈**：❌ 小紅書**沒有穩定發佈 API**，postiz 不支援。小紅書只能**產出草稿**讓使用者手動貼。S5 對小紅書只給「草稿 + 發文建議」，不承諾自動發。

對其他平台同理：搜不到夠好的爆款時，主動請使用者貼幾篇他覺得好的範例，比硬搜低品質結果強。

## 輸出格式（S2 報告）

```markdown
# S2 社群素材挖掘: {方向}

## 爆款範本卡
{逐張卡，至少跨 2-3 平台}

## 跨平台共同模式
- 最易爆平台:
- 最強 hook 骨架（2-3 個）:
- 已寫爛的角度（避）:
- 你的縫隙（沒人好好寫的）:
- 各平台格式差異:

## 證據需求
{若需事實 → 已叫 longform-collect? Material Cards 摘要}

## 建議下一步
進 S3：把你的核心觀點焊進「{選定角度}」+「{最強 hook 骨架}」
```

## 規則
- 偷結構句式，不偷內容；把爆款反向工程成填空骨架
- 至少跨 2-3 個目標平台挖，別只看一個
- 互動數據能填就填，填不了標「概估/未知」，不要編
- 搜不到好範例就請使用者貼，別用爛素材硬湊
- 小紅書/任何登入牆平台：挖料可、自動發不可，說清楚
