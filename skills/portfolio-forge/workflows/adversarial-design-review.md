# 階段 3:對抗式設計評審(靈魂)

這是這份 skill 和「隨便生生圖就複刻」的分水嶺。參考圖生出來**不是拿去複刻,是拿去挑戰**。原文四步法沒有這一步,結果就是「看起來像但字級爆掉、一堆幻覺、沒有呼吸感」。

## 心態

你是一個**挑剔的藝術總監**。設計師交來一疊參考圖,你的工作是找出每張**為什麼還不能用**。不留情面,但每個意見都要具體、可執行(「這裡不行」要接「應該怎麼改」)。

## 兩層評審

### 第一層:Claude 自審
逐張 `Read` 參考圖,對每張跑下面的挑戰清單,先過濾掉明顯問題。

### 第二層:交還使用者
把圖打開給使用者看(`open refs/` 開 Finder),整理你的觀察 + 亮點,請使用者逐張給意見。**使用者的審美是最終標準**,你的自審是幫他省時間。

## 六大挑戰維度(逐張問)

### 1. 字級失控?
- 主視覺大字有沒有溢出版面 / 淹進照片 / 撞到其他元素?
- (實戰最常見:hero 名字太大,最後一個字沉進右側照片,對比失控)
- 修法:字級縮小、完整落在留白欄位內,寧可小一點靠留白撐氣場。

### 2. 沒有呼吸感?
- 文字是不是塞滿整段、像沒排版過的 word 檔?
- (實戰最常見:About 三段長文塞滿,浪費既有美感)
- 修法:長句砍成主句 + 關鍵字標籤 + 一短句,留白加倍。

### 3. 構圖落入 AI 套路?
- 是不是每張都「左字右圖」或每張都置中?
- imagegen skill 本來就要求構圖多樣,檢查它有沒有做到。
- 修法:要求不同 section 用不同構圖(出血、置底、編輯式錯位…)。

### 4. 有幻覺文字?
- 圖裡有沒有 AI 亂生的字、跟 CONTENT.md 不符的文案、書頁/截圖裡的亂碼?
- 這些複刻時**一律以 CONTENT.md 為準**,但要標記出來提醒複刻階段替換。

### 5. 跨圖不一致?
- N 張圖是不是同一套色盤、同一種字型氣質、同一種版式語彙(章節編號格式、分隔線)?
- (實戰常見:各張角落頁碼標記對不上 —— 複刻時統一即可,不必為此重生。)

### 6. 素材呈現對不對?
- 該用真實照片的地方,參考圖是不是只是「意象佔位」?(是,之後要換真的)
- Projects 卡片截圖佔位有沒有預留正確版位?

## 決策

對每張圖:

```
第 X 張 [section 名]:
判定:[通過 / 重生]
亮點:[做對了什麼,保留]
問題:[具體哪裡不行]
修改方向:[怎麼改 —— 給複刻或重生用的明確指令]
```

- **通過** → 定稿,列入複刻素材。
- **重生** → 把「修改方向」寫成明確 prompt,回 `reference-image-gen.md` 只重生這張。

## 迭代紀律

- **會來回幾輪是正常的**,不要為了快跳過。這一步的每一輪都比複刻後才發現問題便宜十倍。
- 重生的回饋要具體到「字級縮小、不疊照片」這種程度,不要只說「不好看」。
- 使用者說某張 OK 就定稿,不要過度優化他已經滿意的。
- 全部定稿後,把要點回寫進 CONTENT.md(如「Hero v2:名字縮小不疊照片」「About v2:精簡要呼吸感」),讓複刻階段有依據。

## 產出

- 定稿的 `refs/section-*.png`(舊版備份為 `refs/old-section-*.png`)
- CONTENT.md 更新(記錄各 section 的定稿要點)

完成後進入 `frontend-replication.md`。
