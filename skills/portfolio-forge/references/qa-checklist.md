# 逐段驗收清單 QA Checklist

複刻完成後,用 puppeteer 逐 section 截圖,對照參考圖,用這份清單驗收。**每個 section 都要過**,不到位退回複刻階段迭代。

## 逐 section 通用檢查

對每一段截圖,對照同段參考圖問:

- [ ] **布局吻合**:主要元素位置、比例和參考圖一致?
- [ ] **配色吻合**:底色 / 文字色 / 強調色和 CONTENT.md 定義一致?強調色只用了那一種?
- [ ] **字型吻合**:英文襯線、中文字型有正確載入(不是 fallback)?字級層次對?
- [ ] **間距/呼吸感**:留白足夠?沒有擠成一團?
- [ ] **真實素材到位**:照片是本人真照片、截圖是真截圖,不是佔位或 AI 生成?
- [ ] **無幻覺文字**:文案完全來自 CONTENT.md,沒有參考圖帶進來的亂字?

## 專項檢查(對應各 section)

- [ ] **Hero**:名字完整落在版面內,沒淹進照片 / 沒溢出?導覽連結可點?
- [ ] **Projects**:卡片是可擴展 grid?截圖有適當裁切?每張有 live 連結?有「更多專案 →」出口?
- [ ] **Writing**:**沒有裸露 URL**?標題本身是超連結?每項有「閱讀全文 →」?
- [ ] **CTA**:主 CTA 按鈕連對地方(訂閱/聯絡)?
- [ ] **Footer**:每個社群連結、email(mailto:)實際點驗正確?

## 全站檢查

- [ ] **連結全綠**:`grep href` 抓出所有連結,逐一確認指向正確、可開。
- [ ] **RWD**:390px 寬行動版逐段檢查 —— 單欄降級?hero 照片改上圖下文?文字不溢出?
- [ ] **動畫**:進場動畫有作用?且尊重 `prefers-reduced-motion`?
- [ ] **資產載入**:CSS / JS / 圖片在 dist 都存在,線上版狀態碼全 200?
- [ ] **build 乾淨**:`npm run build` 無錯,dist 產物完整。

## 評分(可選,給多版本比較用)

四維度各 1–5,誠實給分:

- **忠實度**(和參考圖多像):[?/5]
- **質感**(字距/留白/細節打磨):[?/5]
- **完整度**(真實素材、連結、RWD、動畫都到位):[?/5]
- **可維護性**(資料驅動、加內容只改陣列):[?/5]

低於 3 的維度必須退回迭代,不能將就。

## 驗收紀律

- **要真的看圖**,不是看 build 成功就宣稱完成。截圖 Read 進來逐張比對。
- 誠實回報:哪段沒到位就說哪段,不含糊「大致還原」。
- 動畫會拖累截圖(進場前是空白),用真實捲動 + 等待,別用靜態截圖誤判成「空白頁」(踩坑詳見 headless-qa-and-deploy.md)。
