# 設計風格庫 Design Style Library

taste-skill 生參考圖時,**風格是最重要的輸入**。這份庫給階段 0 訪談用:讓使用者從具體風格選,而不是抽象描述。每個風格附色盤 + 字型配方 + 適用定位,選定後寫進 `CONTENT.md` 的設計方向。

用 `AskUserQuestion` 呈現時,盡量給 ASCII mockup preview,讓使用者「看到」而不是「想像」。

---

## 風格一:編輯雜誌風 Editorial Magazine ★ 通用首選

襯線大標 + 大幅留白 + 少量強調色,像個人刊物。最耐看、最能傳達「有品味的個人 IP」。

- **色盤**:米白底 `#FAF7F2` / 墨黑字 `#1A1A18` / 單一強調色(墨綠 `#2F4A3C`、赭紅 `#A94A38`、藍墨 `#2B4570` 擇一)
- **字型**:英文高對比襯線(Playfair Display / Canela 氣質),中文標題思源宋體(Noto Serif TC)、內文思源黑體(Noto Sans TC)
- **語彙**:章節編號(01–04)、細分隔線、網格、出血大圖
- **適用**:知識工作者、寫作者、設計師、顧問、獨立開發者 IP
- **雷區**:別讓每個 section 都置中;強調色只用一種

## 風格二:暗色科技風 Dark Tech

深底 + 螢光強調 + 等寬字點綴,terminal / 開發者氣質。

- **色盤**:深底 `#0D0D0F` / 淺灰字 / 螢光強調(青綠 `#3DF5A0` 或紫)
- **字型**:無襯線 grotesk 標題 + 等寬字(JetBrains Mono)點綴
- **適用**:工程師、AI 研究者、DevTool 作者、想強調「硬核」的人
- **雷區**:對比別過頭傷眼;螢光色只點綴不鋪面

## 風格三:極簡黑白 Minimal Mono

純黑白灰、無裝飾、內容即設計,瑞士 / Berkeley 風格。

- **色盤**:純白 / 純黑 / 一階灰,零強調色或極克制
- **字型**:單一無襯線家族(Inter / Helvetica Now),靠字重與留白分層
- **適用**:設計師、建築、攝影、走「作品說話」路線的人
- **雷區**:極簡不等於空;靠嚴謹網格與字級節奏撐,不然會顯得未完成

## 風格四:玩味插畫風 Playful Illustration

手繪插畫 + 明亮配色 + 圓角,親和有記憶點。

- **色盤**:暖色系 + 高明度,可 2–3 色
- **字型**:圓潤無襯線,標題可用個性 display 字
- **適用**:創作者、教育者、有吉祥物/IP 角色的人、面向大眾
- **雷區**:插畫風格要統一(同一套筆觸/上色);別淪為 clipart 拼貼

## 選色紀律(共通)

- **強調色只用一種**,全站連結、章節編號、CTA 按鈕統一走它。
- 強調色若能呼應使用者照片裡的元素(如庭園綠、衣著色),整站會更協調。
- 底色忌用純白 `#FFFFFF`,暖白 / 米白更有質感、更不刺眼。

## 字型落地紀律

- 中文字型自托管或用 fontsource,避免明顯 FOUT(閃一下無樣式)。
- 英文襯線 + 中文黑體/宋體的混排,注意行高與標點對齊。
- 參考圖用的字型和複刻用的字型要對得上,否則 QA 會發現「感覺不一樣」。
