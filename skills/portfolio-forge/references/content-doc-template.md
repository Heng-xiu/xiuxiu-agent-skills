# 內容文檔模板 CONTENT.md

這份文檔是**整條流水線的唯一真值**。參考圖、複刻、QA 全部以它為準。動手做圖前先寫好、給使用者確認定稿。

存成專案根目錄的 `CONTENT.md`。標註「已定稿」的欄位改動要回頭知會使用者。

## 寫文檔的紀律

- **抓真資料,不腦補**:用 WebFetch 抓使用者既有網站 / Substack / 社群,自我介紹要**引用原文**,不要自己編。
- **定位標籤 vs tagline 分清楚**:overline 是身份標籤(短),tagline 是一句話價值主張(可展開)。
- **About 一定要短**:這是最容易寫爛的區。主句一句 + 關鍵字標籤群 + 一短句就好,留白比文字重要。
- **連結全部填真的**:footer 每個社群連結、每篇文章 URL、每個 live demo,動手前逐一確認可開,不可捏造。
- **標註擴展性**:Projects / Writing 註明「資料驅動、日後追加只改陣列」,複刻時才會做成可維護結構。
